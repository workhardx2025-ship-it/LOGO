# Решение проблем

## Ошибка 500 при загрузке учеников

### Шаг 1: Проверьте, что сервер запущен

Убедитесь, что backend сервер запущен на порту 3001:
```bash
cd server
npm start
```

Вы должны увидеть:
```
Connected to SQLite database
Students table ready
Server running on port 3001
```

### Шаг 2: Проверьте health endpoint

Откройте в браузере: http://localhost:3001/api/health

Должен вернуться JSON:
```json
{
  "status": "ok",
  "database": "connected"
}
```

Если `database: "not connected"`, значит база данных не инициализирована.

### Шаг 3: Проверьте установку зависимостей

Убедитесь, что все зависимости установлены:
```bash
cd server
npm install
```

Особенно важна библиотека `sqlite3`:
```bash
npm list sqlite3
```

### Шаг 4: Проверьте права доступа

Убедитесь, что у приложения есть права на создание файла базы данных в папке `server/`.

### Шаг 5: Проверьте логи сервера

Посмотрите в консоль, где запущен сервер. Там должны быть сообщения об ошибках, если они есть.

### Шаг 6: Удалите и пересоздайте базу данных

Если база данных повреждена:
```bash
cd server
rm database.sqlite  # или del database.sqlite на Windows
```

Затем перезапустите сервер - база данных создастся автоматически.

### Шаг 7: Проверьте переменные окружения

Убедитесь, что файл `server/.env` существует (если нужен для других настроек).

## Другие распространенные проблемы

### Микрофон не работает
- Проверьте разрешения браузера
- Используйте HTTPS или localhost (некоторые браузеры требуют HTTPS)

### OpenAI API ошибки
- Проверьте, что `OPENAI_API_KEY` установлен в `server/.env`
- Убедитесь, что у вас есть кредиты на OpenAI аккаунте

### CORS ошибки
- Убедитесь, что backend запущен на порту 3001
- Проверьте, что frontend использует правильный proxy в `vite.config.js`
